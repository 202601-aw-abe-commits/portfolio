## 1. 企画提案書

### 1-1. プロジェクト概要
- サービス名: カフェナビ
- コンセプト: 学生のためのカフェ検索サービス。
- ターゲットユーザー:
  - 年齢層/ 高校生、大学生
  - 職業/ 学生

### 1-2. 背景と課題
- 市場の現状:
  - 近年、日本のカフェ市場は拡大傾向にあり、カフェ・喫茶店を含む市場規模は約1兆円規模とされている。特にコロナ禍以降、テレワークやフリーランスの増加により、カフェを「作業場所」として利用する人が増えている。
  - またWi-Fiや電源コンセントを備えたカフェ、長時間滞在が可能なカフェなど、作業環境を重視した店舗の需要も高まっている。
  - 一方で、作業に適したカフェを探す際には「Wi-Fiの有無」「電源席の有無」「混雑状況」「長時間利用の可否」などの情報が分かりにくいという課題がある。そのため、作業環境の条件からカフェを検索できるサービスへのニーズが高まっている。
- 解決すべき課題:
  - 作業環境に適した店舗を見つけることの難しさ
  - 「Wi-Fiの有無」「電源席の有無」「席の広さ」「混雑状況」「長時間利用の可否」

### 1-3. 提案するソリューション
- サービスの特長:
  - 1 勉強・作業向けカフェに特化した検索機能
  - 2 作業環境の条件で絞り込み検索が可能
  - 3 学生目線のカフェ情報を掲載

### 1-4. 制作範囲
- Webサイト（フロントエンド）の範囲:
  - 本Webサイトでは、ユーザーが自分の目的や好みに合ったカフェを検索・比較・閲覧できるサービス画面を制作範囲とする。主に、カフェ検索、詳細情報の閲覧、お気に入り保存、口コミ閲覧など、ユーザーが快適に店舗を探せるためのフロント機能を対象とする。
- Webアプリケーション（サーバーサイド）の範囲:
  - 本Webアプリケーションでは、ユーザー認証、会員情報管理、カフェ情報の登録・更新・取得、検索条件に応じた店舗情報の抽出、お気に入り管理、口コミ・レビュー投稿機能を実装する。さらに、これらの情報をデータベースで一元管理し、フロントエンドへ適切に提供するサーバーサイド処理を構築する。

### 1-5. 使用技術
- フロントエンド: HTML・CSS・JavaScript
- CSSフレームワーク: （Tailwind CSS 等）
- バックエンド: Java / Spring Boot
- データベース: （MySQL / H2 等）
- デザインツール: Figma
- バージョン管理: Git / GitHub
- AI活用: ChatGPT / Claude

### 1-6. スケジュール
- 2週間: 企画・設計
- 2〜3週間: デザイン制作（サイトデザイン、UI設計）

## 2. マーケットリサーチ

### 2-1. 3C分析
- Customer（顧客）:
  - カフェナビのターゲット顧客は、リモートワーカーや学生、副業・自己学習を行う社会人など、カフェを「飲食」ではなく作業場所として利用する人たちである。彼らはWi-Fiや電源、静かさといった作業環境の質を重視し、現在地から条件に合うカフェを素早く見つけたいというニーズを持つ。また、作業に適しているかどうかを一目で判断できる分かりやすい情報や、信頼できる最新のレビューを求めている。行動としては、GoogleマップやSNSなどで検索し、その場で最適な場所を選ぶ傾向が強い。一方で、実際に行ってみないと分からない・情報が古いといった課題を感じており、正確で実用的な情報へのニーズが高い。したがって、作業環境に特化した検索性と情報の分かりやすさが重要となる。
- Competitor（競合）:
  - 競合サービスであるGoogleマップや食べログなどは、掲載店舗数の多さや知名度の高さ、口コミの豊富さが強みであり、ユーザーが日常的に利用している点で非常に優位である。一方で、カフェを「作業場所」として探すための情報（Wi-Fi・電源・静かさなど）が整理されておらず、目的に合った店舗を見つけにくいという弱みがある。また、レビューも飲食中心で作業視点の情報が少ないため、実際に行ってみないと分からないケースが多い。そのため、作業用途に特化した検索性や情報の分かりやすさという点では不十分であり、ここにカフェナビが差別化できる余地がある。
- Company（自社）:
  - カフェナビ（自社）の強み・提供できる価値は、ユーザーが「今行きたいカフェ」をスムーズに見つけられる点にあります。位置情報や利用シーン（作業・デート・一人時間など）に応じて最適なカフェを提案できるため、探す手間を大幅に削減できます。さらに、雰囲気や設備（Wi-Fi・電源・静かさなど）といった細かなニーズにも対応できる点が強みです。口コミや写真を通じてリアルな情報を得られることで、来店前の不安を軽減します。加えて、個々の好みに合わせたレコメンド機能により、新しいカフェとの出会いも提供できます。結果として、ユーザーのカフェ体験そのものの満足度を高める価値を持っています。

### 2-2. SWOT分析
- Strengths（強み）:
  - 1 シンプルで使いやすいUI設計ができる
  - 2 カフェ利用経験があり、ユーザー目線でニーズを理解している
  - 3 作業環境（Wi-Fi・電源・静かさなど）をスコア化する独自の差別化がある
- Weaknesses（弱み）:
  - 1 店舗情報の更新・管理に手間がかかる
  - 2 マーケティング実務の経験が不足している
  - 3 資金やリソースが限られている（低予算運営）
- Opportunities（機会）:
  - 1 リモートワークやオンライン学習の普及により、作業できるカフェ需要が増加している
  - 2 位置情報技術の発展により、現在地からの店舗検索ニーズが高まっている
  - 3 副業・フリーランス増加により、カフェ作業ユーザーが増えている
- Threats（脅威）:
  - 1 Googleマップや食べログなどの大手競合が既に存在している
  - 2 カフェ情報の変化（営業時間・閉店など）により情報が古くなりやすい
  - 3 SNSやブログで情報収集するユーザーが増え、検索サービスを使わない可能性がある

### 2-3. STP分析
- Segmentation（市場細分化）:
  - まず「利用目的」で分けると、作業・勉強目的、休憩・雑談目的、デート・雰囲気重視などに分類できる。次に「ライフスタイル・職業」では、リモートワーカー、フリーランス、学生、会社員などに分けられる。また「利用シーン・状況」として、外出先で急いで探す人、長時間滞在したい人、移動中に一時利用する人などの違いがある。さらに「重視する条件」によって、Wi-Fi・電源重視、静かさ重視、立地重視などの軸でも分類可能である。これらの基準により、カフェ利用者を多角的に細分化することができる。
- Targeting（ターゲット選定）:
  - カフェナビのターゲットは、「作業・勉強目的でカフェを利用するリモートワーカー、フリーランス、学生などの層」に設定する。このセグメントはWi-Fi・電源・静かさといった明確なニーズを持っており、一般的なカフェ検索サービスでは十分に満たされていないため、差別化しやすい。また、リモートワークや副業の普及により市場自体が拡大しており、継続的な利用も見込める。さらに、利用頻度が高くリピート性もあるため、サービスの成長や口コミ拡散にもつながりやすい点から、このセグメントを優先的に狙うべきである。
- Positioning（ポジショニング）:
  - カフェナビは、Googleマップや食べログのような「網羅型の店舗検索サービス」に対して、「作業環境に特化したカフェ検索サービス」としてポジショニングを取る。競合が飲食や立地、口コミの量を重視しているのに対し、カフェナビはWi-Fi・電源・静かさ・作業しやすさといった実用的な指標を可視化し、ユーザーが一目で判断できる点で差別化する。また、作業目的のユーザーに特化することで、専門性と検索精度の高さを強みとし、「作業できるカフェを探すならこれ」という明確なポジションを確立する。

### 2-4. 4P / 4C分析
- Product（製品）:
  - 勉強・作業に適したカフェを検索できるWebサービス。
  - Wi-Fi・電源・席の広さ・静かさ・長時間利用の可否などの条件で絞り込み検索が可能。
- Customer Value（顧客価値）:
  - 勉強・作業に適したカフェを効率よく見つけられる。
  - 自分の目的に合った環境のカフェを事前に確認できる。
- Price（価格）:
  - 基本利用は無料。
  - 将来的に店舗掲載の有料プランや広告掲載で収益化を検討。
- Cost（顧客コスト）:
  - サービス利用は無料。
  - カフェ利用料金のみが発生するため低コストで利用できる。
- Place（流通）:
  - Webブラウザで利用可能。
  - スマートフォン・PCからアクセスでき、外出先でも利用できる。
- Convenience（利便性）:
  - スマートフォンから簡単に検索可能。
  - 条件検索により自分に合ったカフェを短時間で見つけられる。
- Promotion（販促）:
  - SNS（Instagram・TikTok・X）での情報発信。
  - SEO対策による検索流入。
  - 学生コミュニティや大学周辺での認知拡大。
- Communication（対話）:
  - SNSやレビューを通じてユーザーと情報共有が可能。
  - 利用者の口コミや評価を参考にカフェを選べる。

## 3. ペルソナシート

### 3-1. 基本プロフィール
- ペルソナ基本情報: 鈴木健太（２１）

### 3-2. ライフスタイル
- 平日のタイムスケジュール:
  - 8:00 起床・通学準備
  - 9:00〜12:00 大学の授業
  - 12:00〜13:00 学食またはカフェで昼食
  - 13:00〜16:00 授業・空き時間に課題やレポート作業
  - 16:00〜18:00 カフェでPC作業（レポート・調べ物）
  - 18:00〜20:00 アルバイト（飲食店）
  - 21:00 帰宅・夕食
  - 22:00〜24:00 スマホやPCでSNS・動画視聴・課題
- 休日の過ごし方:
  - 自宅だけでなく気分転換を兼ねてカフェで過ごすことが多い。午前中は比較的静かなカフェを探して資格勉強や副業の作業を行い、集中できる環境を重視して場所を選ぶ。昼頃には軽く食事をとり、そのまま数時間作業を続けることもあるが、混雑してきた場合は別のカフェへ移動するなど、場所を変えながら効率よく時間を使う。作業の合間にはスマホで次の作業場所を探したり、SNSやレビューをチェックするなど、常により良い環境を求めて行動している。夕方以降は作業を切り上げ、リラックス目的で別のカフェや自宅でゆっくり過ごすことが多い。
- 趣味・関心事:
  - カフェ巡り
  - YouTube・動画視聴
  - SNS閲覧（トレンド・カフェ情報）
  - 友人との食事
  - ガジェットや便利アプリ

### 3-3. デジタル行動
- 使用デバイス:
  - iPhone（メイン端末）
  - ノートPC（大学課題・作業用）
  - ワイヤレスイヤホン
- 使用サービス:
  - Instagram（カフェ情報・友人との交流）
  - Googleマップ（カフェ検索）
  - YouTube（情報収集・娯楽）
  - LINE（連絡）
  - Notion / Googleドキュメント（課題・メモ）

### 3-4. 課題とニーズ
- 抱えている課題・悩み:
  - 作業や勉強に適したカフェを探すのに時間がかかる。
  - Wi-Fiや電源、席の広さ、静かさなど作業環境の情報が事前に分かりにくい。
  - 実際に行ってみないと作業しやすいか判断できない。
- サービスに期待すること:
  - Wi-Fi・電源・静かさ・長時間利用可など作業環境の条件でカフェを簡単に検索できる。
  - 事前に作業しやすい環境が分かり、無駄な移動や時間を減らせる。
  - 自分の用途（勉強・レポート・PC作業）に合うカフェをすぐ見つけられる。

### 3-5. カスタマージャーニーマップ
- 認知:
  - SNSや検索で「勉強できるカフェ」「作業カフェ」などを検索する友人やSNS投稿で作業向けカフェの情報を知る大学や街中でカフェ情報を見かける
- 興味:
  - Wi-Fi・電源があるカフェを探し始める
  - 作業しやすい環境のカフェがあることに興味を持つ
  - サイトやSNS投稿を見て「便利そう」と感じる
- 比較検討:
  - Googleマップやカフェレビューサイトと比較する
  - Wi-Fi、電源席、席の広さ、静かさなどの条件を確認する
  - 口コミや評価をチェックする
- 行動:
  - サイトで条件検索をしてカフェを探す
  - 気になったカフェを保存または訪問する
  - 実際にそのカフェに行き作業・勉強をする
- 共有・継続:
  - SNSでカフェ情報をシェアする
  - レビューや口コミを投稿する
  - 友人におすすめの作業カフェとして紹介する
