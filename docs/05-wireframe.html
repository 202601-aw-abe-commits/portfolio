1 ワイヤーフレーム作成方法

2 トップページ

トップページ ワイヤーフレーム（PC版） 必須
Figmaで作成したワイヤーフレームのスクリーンショットを配置

配置意図の説明 必須

1. ヘッダー（ロゴ・ナビゲーション）

配置理由

  • 画面上部に固定配置することで、ユーザーがどのページにいても主要機能へすぐアクセスできるようにしている。
  • 左側にロゴを配置することでサービス認知を高め、「学生向け作業カフェ検索サービス」であることを直感的に伝えている。
  • 「検索」を中央付近で強調することで、サービスの主目的を明確化している。

ユーザーへの効果

  • 初めて利用するユーザーでも迷わず操作できる。
  • 「お気に入り」「お問い合わせ」などへすぐ移動でき、回遊性が向上する。
  • サービスの専門性（作業カフェ特化）が認識しやすくなる。

2. メイン検索バー（検索入力＋現在地検索）

配置理由

  • ファーストビュー中央に大きく配置し、最優先行動である「カフェ検索」をすぐ実行できるようにしている。
  • 「現在地から探す」ボタンを右側に配置することで、スマホ利用時でも片手操作しやすい導線にしている。
  • 背景画像の上に配置することで、カフェの雰囲気を視覚的に伝えながら検索体験を印象づけている。

ユーザーへの効果

  • 「今すぐ作業できるカフェを探したい」というニーズに即対応できる。
  • 入力負荷を減らし、検索までの時間を短縮できる。
  • 現在地検索により外出先でも素早く店舗を探せる。

3. 絞り込み条件サイドバー

配置理由

  • 左側に固定配置することで、検索結果を見ながら条件変更しやすいUIにしている。
  • 「Wi-Fi」「電源」「静か」「長時間OK」など、作業ユーザーが重視する条件を一覧化して視認性を高めている。
  • 検索結果より先に条件が目に入る構成にすることで、「作業環境特化サービス」という差別化を強調している。

ユーザーへの効果

  • 自分に合ったカフェを効率よく絞り込める。
  • Googleマップや食べログでは探しにくい「作業環境」を簡単に比較できる。
  • 無駄な移動や検索時間を削減できる。

4. 検索結果表示エリア

配置理由

  • 画面中央〜右側の広い領域を使い、複数のカフェ情報を一覧比較しやすくしている。
  • 「11件のカフェが見つかりました」を上部に表示することで、検索結果の安心感と検索成功を明示している。
  • カード型レイアウトを採用し、スマホ・PCどちらでも視認しやすい構成にしている。

ユーザーへの効果

  • 複数店舗を直感的に比較できる。
  • 情報が整理されており、短時間で意思決定しやすい。
  • 視覚的に見やすいため、ストレスなく閲覧できる。

5. カフェカード（画像・スコア・設備情報）

配置理由

  • カフェ画像を大きく表示することで、店舗の雰囲気を直感的に伝えている。
  • 「作業スコア」を左上に配置し、作業向けカフェであるかを一目で判断できるようにしている。
  • Wi-Fi・電源・静かさなどをタグ形式で表示し、必要情報を即確認できるようにしている。
  • 営業時間や距離を下部にまとめ、比較しやすくしている。

ユーザーへの効果

  • 「作業しやすいか」を瞬時に判断できる。
  • 店舗詳細を開かなくても必要情報を把握できる。
  • 作業目的ユーザーに最適化された比較体験を提供できる。

6. お気に入りアイコン

配置理由

  • カード右上に配置し、視線の流れの中で自然に保存操作できるようにしている。
  • 店舗情報を閲覧しながら即保存できるUI設計にしている。

ユーザーへの効果

  • 気になる店舗を後から見返しやすい。
  • 比較検討中のユーザー体験を向上できる。
  • リピート利用や継続利用につながる。

7. 背景ビジュアル

配置理由

  • カフェ写真を背景に使用することで、サービス全体の世界観を演出している。
  • 透明感のあるレイヤーを重ねることで、情報の視認性を保ちながら雰囲気を表現している。

ユーザーへの効果

  • 「カフェで作業するイメージ」を想起しやすい。
  • サービス利用時の期待感や没入感を高める。
  • 視覚的な印象を強化し、記憶に残りやすくなる。

全体のUI配置意図

本画面は、「作業・勉強向けカフェを素早く探したい学生ユーザー」を対象に、
“検索 → 条件絞り込み → 比較 → 保存” を最短で行える導線を意識して設計している。

特に、企画段階で定義された課題である
「Wi-Fi・電源・静かさなどの情報が探しにくい」という問題に対し、
条件検索と作業スコアを視覚的に分かりやすく配置することで、
一般的なカフェ検索サービスとの差別化を図っている。

3 下層ページ

下層ページ ワイヤーフレーム 必須
各下層ページのワイヤーフレームを順に配置

【ページ名1】

配置意図：

このページ全体の配置意図

このページは、
「検索結果が0件だった場合でも、ユーザーを離脱させず再検索へ誘導すること」
を目的として設計されている。

通常のエラー画面ではなく、

  • 検索条件の見直し
  • エリア変更
  • 条件リセット
  • トップへ戻る

といった複数の行動導線を残すことで、
ユーザーが「次に何をすればよいか」を直感的に理解できる構成にしている。

また、企画資料で定義されている
「作業環境に合うカフェを効率的に探したい」という課題に対し、
検索失敗時でもスムーズに再挑戦できるUXを意識している。

【ページ名2】

配置意図：

このページ全体の配置意図

このページは、
「初めてサービスを利用するユーザーに、カフェナビの目的・特徴・使い方を分かりやすく伝えること」
を目的として設計されている。

特に、

  • 作業向けカフェ検索に特化していること
  • Wi-Fiや電源など条件検索ができること
  • お気に入り保存で効率的に利用できること

を、文章だけではなく
タグ・カード・イラスト・UIイメージを使って視覚的に説明することで、
ユーザーが短時間でサービス価値を理解できる構成になっている。

また、企画資料で定義された
「作業しやすいカフェを効率よく探したい」という課題に対し、
検索から保存までの利用フローをシンプルに伝えるUXを意識している。

配置意図：

このページ全体の配置意図

このページは、
「ユーザーが迷わず安心して問い合わせできること」
を目的として設計されている。

特に、

  • 必要項目を分かりやすく整理
  • 入力しやすいフォーム設計
  • 行動導線を明確化
  • 安心感を与えるデザイン

を意識し、
問い合わせ時の心理的負担を減らすUI構成になっている。

また、企画資料で定義されている
「ユーザー体験を向上させるサービス」という目的に合わせ、
不具合報告や要望を簡単に送れることで、サービス改善につながるUXを意識している。
