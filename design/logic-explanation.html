IF-THEN-ELSE形式でロジックを記述
IF ログイン画面で送信イベントが発生した THEN
    email と password を取得する
    /api/login へ POST する

    IF 通信エラーが発生した THEN
        「通信エラー」メッセージを表示する
    ELSE
        IF レスポンスステータスが 200 以外 THEN
            エラーメッセージを表示する
        ELSE
            sessionToken を localStorage に保存する
            成功メッセージを表示する
            index.html へリダイレクトする
        ENDIF
    ENDIF
ENDIF

IF /api/login を受信した THEN
    リクエストから email と password を取得する
    email を正規化する
    ※ trim と lowercase を行う

    users テーブルから
    email が一致し、かつ is_active = 1 のユーザーを検索する

    IF ユーザーが存在しない THEN
        401 Unauthorized を返却する
    ELSE
        IF verify_password(password, password_hash) が false THEN
            401 Unauthorized を返却する
        ELSE
            session_token を生成する
            expires_at を14日後に設定する
            users.last_login_at を更新する
            user_sessions にセッション情報を保存する
            200 OK と sessionToken と user情報を返却する
        ENDIF
    ENDIF
ENDIF

バリデーション処理のロジック
IF タスク登録フォームが送信された THEN
    title, description, done を取得する

    IF title が null または 空文字 THEN
        エラーメッセージを設定する
        入力画面を再表示する
    ELSE
        確認画面用に title, description, done を Model に設定する
        確認画面を表示する
    ENDIF
ENDIF

IF 確認画面で登録確定された THEN
    title を受け取る

    IF title が null または 空文字 THEN
        エラーメッセージを設定する
        入力画面を再表示する
    ELSE
        Todo オブジェクトを作成する
        title を設定する
        completed を false で設定する
        DB に登録する
        一覧画面へリダイレクトする
    ENDIF
ENDIF

SET sql = "SELECT * FROM todo WHERE 1=1"
SET params = empty list

    sql += " AND (title LIKE ? OR description LIKE ?)"

    params += "%" + keyword + "%"

IF categoryId is not null THEN
    sql += " AND category_id = ?"
    params += categoryId

IF status is not null AND status is not empty THEN
    sql += " AND status = ?"
    params += status

データアクセス制御のロジック 必須
自分のタスクのみ操作可能にする制御
IF リクエストを受信した THEN
    token = Authorizationヘッダー or Cookie から取得

    IF token がない THEN
        401 Unauthorized を返す
        END
    ENDIF

    session = user_sessions から token で検索（かつ expires_at > 現在時刻）
    IF session がない THEN
        401 Unauthorized を返す
        END
    ENDIF

    loginUserId = session.user_id
    targetTaskId = パスパラメータから取得

    task = tasks から targetTaskId で検索
    IF task がない THEN
        404 Not Found を返す
        END
    ENDIF

    IF task.user_id != loginUserId THEN
        403 Forbidden を返す
        END
    ENDIF

    -- ここまで通過したら本人データなので許可
    更新/削除/詳細取得を実行する
    200 OK（またはリダイレクト）を返す
ENDIF

SQL条件で制御する場合は、更新・削除を WHERE id = :taskId AND user_id = :loginUserId にして、更新件数0件なら権限なしとして扱います。

遷移可否の判定ロジック 必須
状態遷移が許可されるかの判定処理
-- 現行実装（completed: true/false）の遷移可否判定

IF ステータス変更リクエストを受信した THEN
    task = IDでタスクを取得する

    IF task が存在しない THEN
        変更不可（false を返す / エラー応答）
    ELSE
        -- 現行は toggle 方式のため、未完了⇔完了は常に許可
        IF task.completed == true THEN
            task.completed = false
        ELSE
            task.completed = true
        ENDIF

        保存に成功したら変更可（true）
        保存に失敗したら変更不可（false）
    ENDIF
ENDIF

補足: 現行コードでは「最終状態で遷移不可」などの業務制約はなく、存在するタスクであれば常に切替可能です。
